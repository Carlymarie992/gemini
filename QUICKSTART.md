# Quick Start Guide

## Getting Started in 5 Minutes

### 1. Install and Run
```bash
npm start
```

### 2. Set Up Your Case
Select option **2** to update case information:
- Enter your case number
- Enter plaintiff name (you)
- Enter defendant name
- Enter court name
- Enter judge name
- Enter filing date

### 3. Learn About DARVO
Select option **6** to view the DARVO tactics guide
- Understand what DARVO means
- Learn to recognize the tactics
- Understand why documentation matters

### 4. Start Documenting

#### Document a Timeline Event (Option 3)
Record important events in chronological order:
- Date of the event
- What happened (be specific and factual)
- Where it happened
- Who witnessed it

#### Document DARVO Incidents (Option 5)
When the abuser uses manipulation tactics:
- Date and context
- What they denied
- How they attacked you
- How they reversed victim/offender roles
- Who witnessed it

#### Add Evidence (Option 4)
Track all evidence:
- Type (document, photo, video, audio, witness statement)
- Description
- Date obtained
- Where it's stored
- Additional notes

### 5. Prepare for Court
Select option **9** for the complete checklist
- Review what to do before court
- Learn what to do during testimony
- Understand what to watch for during proceedings

### 6. Export Your Records
Select option **10** to create a text file summary
- All case information
- Timeline of events
- Evidence list
- DARVO incidents
- Notes

## Tips for Success

### Be Consistent
- Document events as soon as they happen
- Update your records daily if needed
- Review your timeline regularly

### Stay Factual
- Write exactly what happened, not how you felt
- Use specific dates and times
- Name witnesses whenever possible
- Avoid emotional language in documentation

### Organize Evidence
- Keep physical evidence in a safe location
- Make backup copies of digital evidence
- Record where everything is stored
- Take photos of physical evidence

### Pattern Recognition
- Document every DARVO incident
- Note similarities between incidents
- Show how tactics repeat over time
- Let the pattern speak for itself

### Before Every Court Date
1. Export a case summary
2. Review all documentation
3. Print necessary copies
4. Practice your testimony
5. Review the court preparation checklist

## Common Scenarios

### Scenario 1: First Time Using the Tool
1. Set up case information (option 2)
2. Read DARVO guide (option 6)
3. Add past events to timeline (option 3)
4. Start documenting new incidents as they occur

### Scenario 2: Court Date Coming Up
1. Review all records (option 8)
2. Export case summary (option 10)
3. Review court preparation checklist (option 9)
4. Add any new evidence (option 4)
5. Practice testimony with your timeline

### Scenario 3: After a DARVO Incident
1. Document immediately (option 5)
2. Note the date, time, and context
3. Record all three tactics (deny, attack, reverse)
4. List witnesses
5. Add related evidence

### Scenario 4: Organizing Evidence
1. Add each piece of evidence (option 4)
2. Be specific about type and description
3. Record exact location
4. Cross-reference with timeline events
5. Note related DARVO incidents

## Safety and Privacy

### Keep Data Secure
- The `.case-data` folder contains all your information
- Back up this folder regularly
- Consider encrypting sensitive files
- Store backups in a secure location

### Who Should See Your Records
- Your attorney
- Court officials (through proper filing)
- Trusted advocates

### Who Should NOT See Your Records
- The defendant or their representatives
- Social media or public forums
- People who might share with the defendant

## Getting Additional Help

### Legal Support
- Contact legal aid organizations
- Consult with attorneys experienced in abuse cases
- Ask about protective orders if needed

### Emotional Support
- National Domestic Violence Hotline: 1-800-799-7233
- Local domestic violence organizations
- Support groups for survivors
- Counseling services

### Technical Help
- Keep backups of your data
- If the program has issues, your data is in `.case-data/case.json`
- You can manually edit this file if needed
- Export summaries regularly as backup
